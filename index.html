On the glass

Everything that matters, at a glance

Laid out like a race dash so you read it in a fraction of a second — no menu diving, no scrolling telemetry screens.

Top strip

Shift-light RPM

A green→amber→red→violet rev strip with a redline flash, plus a live RPM readout. Reads ESC electrical RPM and scales it to real prop RPM.

Center

Amps + fuel + pit timer

Live current as the hero number, a fuel gauge that counts mAh remaining once you set your pack size, and a big countdown timer (Timer 1) that runs negative past zero.

Rails

Battery & signal

Segmented battery rail on a real LiPo discharge curve (weakest cell), and a signal rail that only goes live once telemetry is actually flowing.

Per-cell row

4-cell health + resistance

With a SMART ESC: each cell's voltage, internal resistance and health color, weakest cell ringed. Auto-hides on planes that only report pack voltage.

Bottom cluster

Six analog gauges

Volts, cell balance, ESC temp, mAh, peak amps, and pack health (mΩ/cell) — old-school needles that light up amber/red when a reading goes out of range.

Alarms & trims

Voice, buzz & trim tabs

Escalating low-cell alarms (buzz + tone), link-hold warnings, a brownout banner, and graduated trim tabs on all four edges with fine center marks.

Will it work with my receiver?

The same widget, tuned to what your RX reports

FLTHUD flexibly matches dozens of sensor names. The more your receiver streams, the more of the dashboard lights up — and anything missing simply reads -- or hides itself. Here's what to expect by system.

Full dashboard

Spektrum — AR631 / AR631Plus + SMART ESC

The flagship setup · every gauge alive
  • Cel1–Cel4 / CelsPer-cell + pack voltage → 4-cell row, battery rail, cell balance & health, low-cell voice alarm.
  • ECUR / BCurCurrent → amps hero, watts, peak amps, mAh burned, and pack internal resistance.
  • ErpmMotor RPM → shift-light strip + RPM readout (÷10 from electrical RPM).
  • TFET / BTmpESC / battery temp → temperature gauge with caution + critical thresholds.
  • FdeA/B/L/R · FLss · HoldFlight-log link health → signal meter (frame-loss rate) + hold alarm.

Heads-up: the AR631's gyro/accel is used internally for AS3X/SAFE and is not sent as telemetry. Spektrum-over-multiprotocol RSSI reads 0, so the link meter is driven by the frame-loss rate instead — a real, recovering signal-quality gauge.

Grows with sensors

FrSky — ACCST (D16) / ACCESS

Archer, SR10+, etc. · real RSSI out of the box
  • RSSILink quality → drives the signal rail directly (FrSky reports true RSSI).
  • RxBtRX / servo-rail voltage → BEC rail readout (this is not your pack voltage on a bare RX).
  • + FASAdd a current sensor → unlocks the battery gauge, watts, peak amps and pack resistance.
  • + FLVSSAdd a cell sensor → lights up the per-cell row and balance/health.
  • + GPSAdds speed / altitude / sats (the widget already has slots wired for these).

A bare stabilization RX (e.g. Archer Plus SR10+) wakes the HUD on RSSI alone. For the full power dashboard, add an S.Port FAS/FLVSS or GPS.

Bare bones

FlySky — AFHDS / AFHDS2A

Micros & budget RXes · link + whatever it reports
  • RSSI / RQlyLink quality → signal rail and link-loss alarms.
  • RxBt / VoltVoltage if the RX reports it → battery rail + VOLT gauge.
  • + sensorCurrent / RPM / temp need an add-on FlySky or generic sensor — add them and those gauges come alive.

On a plain FlySky RX you'll get the link meter, timer, RPM strip idle, and any voltage it reports. Everything else stays hidden until a sensor feeds it — nothing looks broken.

Any EdgeTX color radio

Anything else

Generic / mixed setups
  • VFAS · Curr · RPM · Tmp1Standard EdgeTX sensor names are all matched — most generic setups just work.
  • mAh / Fuel / CapaA real capacity sensor is used directly; otherwise mAh is integrated from current.
  • Rule of thumb: if EdgeTX can discover the sensor, FLTHUD will try to use it — and hide the gauge if it can't.

Because every element is data-driven, one widget covers a whole hangar of mixed protocols. Set it once, fly anything.

Step by step

Installing FLTHUD on your TX16S

Five minutes with the SD card. This is a widget for color radios — not a Lua telemetry script — so it lives in /WIDGETS/, not /SCRIPTS/.

1 · Download the widget

Grab main.lua from the button below (or this page's download section). It's a single ~36 KB text file.

2 · Put it on the SD card

Connect the radio to your PC and choose USB Storage (SD) so the card mounts as a drive. On the card, open /WIDGETS/, create a folder named FLTHD2 (exact spelling), and drop the file in as:

SD:/WIDGETS/FLTHD2/main.lua

Updating an existing install? Also delete any main.luac in that folder — EdgeTX caches a compiled copy and will otherwise load the old build.

3 · Eject & reboot

Safely eject the card and unplug, then power the radio normally. USB storage mode and the widget can't run at the same time, so you must leave storage mode to see it.

4 · Discover your sensors

Power your model so it's transmitting telemetry, then on the radio go Model → Telemetry → Discover new sensors. Cloned a model from another plane? Delete all sensors first, then discover — stale sensor names from the old model can mask the real ones.

5 · Put it on a screen

From the main view, long-press the screen (or go Model → Screens). Set a screen's layout to Full screen (1×1), tap the zone, and choose the widget FLTHD2. It draws a 480×272 panel scaled to fill the display.

6 · Set your pack size (optional)

Open the widget's settings and set PackX100 to your pack in units of 100 mAh — 22 for a 2200, 50 for a 5000. Now the fuel gauge shows mAh remaining instead of counting up. Leave it at 0 to just count mAh used.

7 · Pit timer (optional)

The big countdown reads Timer 1 — set that up (e.g. a throttle-gated count-down) and it drives the center clock, going negative past zero so you know how far over you ran.

Want to see it before you fly? Open main.lua in any text editor and set DEMO = true near the top (line ~11), reinstall, and the dashboard animates a simulated flight on the bench — exactly like the preview on this page. Set it back to false for real telemetry.
